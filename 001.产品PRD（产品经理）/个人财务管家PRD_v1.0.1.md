## 2. 产品概述

### 2.1 产品背景

在现代生活中，个人和家庭的经济管理变得越来越重要。很多人在日常生活中缺乏对消费的有效记录和分析，导致月末回顾时发现支出超出预期，却不清楚钱花在了哪里。养成良好的消费记录习惯，可以帮助用户清晰地了解自己的消费情况，从而更好地规划财务、培养理性的消费习惯。

### 2.2 产品简介

个人财务管家（Velvet Ledger）是一款简洁优雅的个人财务管理应用，采用 **Ethereal Minimalism（空灵极简主义）** 设计风格，通过柔和的色彩、毛玻璃效果和半透明层营造温柔的视觉体验。主要帮助用户记录日常消费、分类统计支出、分析消费趋势、设定并管理预算。通过直观的可视化图表和简洁的操作流程，让用户轻松掌握自己的财务状况。

### 2.3 产品定位

- **目标用户**：有日常消费记录需求、关注个人财务状况的用户群体
- **核心价值**：简单、快速地记录消费，提供清晰的统计和分析
- **产品调性**：简洁、优雅、温暖、可信赖
- **设计风格**：Ethereal Minimalism（空灵极简主义）- 柔和、轻盈、质感

### 2.4 核心功能

| 功能模块 | 功能描述 | 页面 |
|----------|----------|------|
| 用户认证 | 用户注册、登录、登出，保护用户数据安全 | login.html, register.html |
| 消费记录 | 快速记录日常支出/收入，包含金额、分类、时间、备注、支付方式 | add-expense.html |
| 分类管理 | 自定义收支分类，支持 Emoji 图标 | categories.html |
| 分类统计 | 按分类维度统计支出分布 | statistics.html |
| 消费趋势 | 通过图表展示消费趋势和变化 | statistics.html |
| 预算管理 | 设定月度预算，追踪预算执行情况 | budget.html |
| 个人中心 | 用户资料管理、账户安全、数据管理、应用设置 | profile.html |

### 2.5 名词解释

- **消费记录**：用户录入的每一笔支出或收入
- **分类**：用于标识消费类型的标签（如餐饮、交通、购物等），支持 Emoji 图标
- **支付方式**：用于支付的手段（微信支付、支付宝、银行卡、现金等）
- **预算**：用户设定的月度消费上限
- **周期**：统计报表的时间范围（本周、本月、近3月、本年等）
- **用户账户**：用户注册后用于登录的账号，包含用户名和密码

## 4. 功能需求

### 4.1 用户认证

#### 4.1.1 用户注册

- 支持输入用户名（3-20个字符）
- 支持输入密码（至少6位）
- 用户名唯一性校验
- 注册成功后自动登录
- 密码使用加密存储（不可明文保存）
- 用户数据存储在 localStorage

#### 4.1.2 用户登录

- 支持输入用户名和密码登录
- 登录失败提示（用户名或密码错误）
- 登录成功后跳转至首页

#### 4.1.3 用户登出

- 清除本地登录状态
- 跳转到登录页面
- 清除本地缓存的财务数据

#### 4.1.4 会话管理

- 登录状态持久化（localStorage）
- 多设备登录支持（数据存储在本地）

---

### 4.2 消费记录

#### 4.2.1 添加消费记录

- 支持输入消费金额，支持小数点后两位
- 支持切换支出/收入类型
- 支持选择消费分类（默认20+种支出分类，10+种收入分类）
- 支持选择消费日期（默认为当天，可修改）
- 支持添加备注（选填，最多100字）
- 支持选择支付方式（微信支付、支付宝、银行卡、现金）
- 保存后显示操作成功提示

#### 4.2.2 编辑消费记录

- 支持修改已有消费记录的所有信息
- 支持删除消费记录（需二次确认）

#### 4.2.3 查询消费记录

- 支持按日期范围筛选（本周、本月、近3月）
- 支持按分类筛选
- 支持按支付方式筛选
- 支持关键字搜索备注
- 记录列表按日期倒序排列
- 显示每日支出/收入/结余汇总

### 4.3 分类管理

- 支出分类：餐饮、购物、交通、娱乐、健身、居住、医疗、教育、通讯、美容、服饰、旅游、社交、宠物、汽车、家居、礼物、零食、数码、其他支出

#### 4.3.2 自定义分类

- 支持新增自定义分类
- 支持选择 Emoji 图标
- 支持添加分类描述
- 支持编辑分类信息
- 支持删除自定义分类

---

### 4.4 分类统计

#### 4.4.1 支出分类占比

- 以环形图展示各分类支出占比
- 显示各分类的具体金额和百分比
- 支持选择时间周期（本周、本月、近3月）

## 5. 非功能需求

### 5.1 性能需求

| 指标 | 要求 |
|------|------|
| 页面加载时间 | 不超过1秒（本地HTML文件） |
| 操作响应时间 | 用户操作后反馈不超过500ms |
| 数据存储 | 支持至少10000条消费记录（localStorage） |
| 离线使用 | 支持，无需网络连接 |

### 5.2 兼容性需求

- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 适配桌面端和移动端屏幕尺寸
- 响应式布局设计
- 深色模式支持（跟随系统）

### 5.3 安全性需求

- 用户密码加密存储（不可逆加密）
- 用户数据仅存储在本地设备
- 不收集用户隐私信息（如通讯录、位置等）
- 支持生物识别解锁（规划中）
- 登录失败提示（防止暴力破解）

### 5.4 可用性需求

- 界面简洁，一目了然
- 操作流程简单，学习成本低
- 错误提示清晰，指导用户正确操作
- 支持离线使用，无需网络连接
- 毛玻璃效果和柔和阴影营造轻盈通透感

### 5.5 可靠性需求

- 数据存储稳定，不丢失（localStorage）
- 备份/恢复功能可靠
- 页面刷新数据不丢失

## 6. UI/UX设计方向

### 6.1 设计理念

- **简洁至上**：减少视觉干扰，让用户专注于内容和操作
- **高效记录**：最大化减少输入步骤，实现快速记账
- **清晰直观**：数据可视化，一眼看清财务状况
- **温暖可信**：配色和设计传达可信赖感
- **空灵极简**：柔和、轻盈、质感的设计语言

### 6.2 设计风格

- **风格**：Ethereal Minimalism（空灵极简主义）
- **配色**：粉色/紫色调的柔和配色，营造温柔视觉体验
- **字体**：Hanken Grotesk（Google Fonts）
- **图标**：Material Symbols Outlined（Google Fonts）
- **质感**：适度使用毛玻璃效果和轻柔阴影，营造轻盈通透感
- **形状**：圆角设计，反映有机自然形态

### 6.3 色彩规范（Velvet Design System）

> 设计灵感来自黄昏时分绽放的花朵和柔软的丝绒质感，传达成熟、宁静、修复的感觉。

| 色彩角色 | 色值 | 用途 |

### 6.5 页面结构

- **登录页**：用户输入用户名和密码进行认证
- **注册页**：新用户注册账号
- **首页**：展示当月概况、预算进度、最近消费记录
- **明细页**：消费记录列表，支持筛选和搜索
- **统计页**：可视化图表展示支出分布和趋势
- **预算页**：预算设置和执行进度
- **分类管理页**：自定义收支分类
- **个人中心页**：设置、数据管理、登出等
- **记一笔页**：快速记账入口

### 6.7 关键交互设计

- **分类选择网格**：Emoji 图标网格便于快速选择
- **滑动操作**：消费记录支持点击编辑/删除
- **编辑弹窗**：点击消费记录弹出编辑弹窗
- **预算进度环**：环形图直观展示预算使用情况
- **筛选器**：日期范围、分类、支付方式筛选

## 7. 技术选型

### 7.1 技术栈

| 类型 | 技术 | 说明 |
|------|------|------|
| 前端框架 | 原生 HTML5 + CSS3 + JavaScript | 无框架，轻量级 |
| CSS 工具 | Tailwind CSS（CDN引入） | 原子化 CSS |
| 图表库 | Chart.js（CDN引入） | 统计图表 |
| 字体 | Hanken Grotesk（Google Fonts） | 西文字体 |
| 中文渲染 | WenQuanYi Zen Hei（系统字体） | 中文字体 |
| 图标 | Material Symbols Outlined（Google Fonts） | 图标库 |
| 数据存储 | localStorage | 本地持久化 |

### 7.2 数据存储

| 方案 | 适用场景 | 特点 |
|------|----------|------|
| localStorage | 所有用户数据 | 轻量、持久、支持 JSON |
| sessionStorage | 临时会话数据 | 页面会话结束清除 |

**存储数据结构：**
- `user`: 用户账户信息
- `records`: 消费记录列表
- `categories`: 自定义分类
- `budgets`: 预算设置
- `settings`: 用户偏好设置

### 7.3 页面文件结构

```
frontend/src/pages/
├── login.html          # 登录页
├── register.html       # 注册页
├── home.html           # 首页（仪表盘）
├── records.html        # 消费明细
├── statistics.html     # 统计分析
├── budget.html         # 预算管理
├── categories.html     # 分类管理
├── profile.html        # 个人中心
└── add-expense.html    # 记一笔
```

### 7.4 状态管理

- 使用原生 JavaScript 管理状态
- localStorage 作为数据持久化层
- 页面间通过 URL 参数传递数据

### 7.5 响应式设计

- 移动端优先设计
- 使用 Tailwind 的响应式前缀（`md:`, `lg:`）
- 移动端显示底部导航，桌面端显示侧边栏
